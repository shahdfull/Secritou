BUG #1: SERVER ERROR MIDDLEWARE
═══════════════════════════════════════════════════════════════════════════

FILE: server/src/middlewares/error.middleware.ts

PROBLEM:
  When multer rejects a file (wrong MIME type, size too large):
  • MulterError or Error + statusCode thrown
  • Error middleware only handled ZodError and HttpError
  • Unhandled errors → Express default handler
  • Default handler returns HTML error page
  • Axios receives HTML, cannot parse, throws generic error

SOLUTION ADDED:
  Line 3: import multer from "multer"

  Lines 8-34: Handle multer.MulterError
    - Map LIMIT_FILE_SIZE → 413
    - Map LIMIT_PART_COUNT, LIMIT_FILE_COUNT → 400
    - Return JSON response instead of HTML
    - Track in metrics

  Lines 36-51: Handle errors with custom .statusCode
    - Catch errors with statusCode property (set by fileFilter)
    - Return JSON with appropriate status
    - Works for 415 (file type not allowed)
    - Track in metrics

RESULT:
  • All errors return JSON (not HTML)
  • Axios can parse response
  • Real error message reaches client
  • Proper HTTP status codes (413, 415)

BUG #2: AXIOS ERROR MESSAGE EXTRACTION
═══════════════════════════════════════════════════════════════════════════

FILE: client/src/api/axios.ts

PROBLEM:
  Server returns: { message: "File type 'application/msword' not allowed..." }
  Axios receives status 415 with JSON body
  Axios wraps everything in AxiosError
  AxiosError.message = "Request failed with status code 415"
  useUpload hook: toast.error(err.message)
  User sees: "Request failed with status code 415" ❌ (not helpful)

SOLUTION ADDED:
  Lines 80-87: Extract real error message at START of error interceptor

  Line 83: Check error.response?.data?.message
    - This is the primary field from server response

  Line 85: Fallback to error.response?.data?.error?.message
    - Alternative structure in case error object nesting

  Line 84 & 86: Set error.message = real server message
    - Now error.message contains "File type... not allowed"

  After this:
    - 401 retry logic runs normally
    - For non-401 errors: Promise.reject(error) with updated message
    - useUpload hook catches error
    - toast.error(err.message) shows real message ✅

RESULT:
  • User sees: "File type 'application/msword' not allowed. Accepted: application/pdf"
  • User sees: "File exceeds maximum size limit"
  • User sees real, actionable error messages
  • No changes to 401/retry logic
